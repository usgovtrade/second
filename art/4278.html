Q: How does the Raspberry Pi store its files? When I do sudo apt-get purge firefox it asks me to save it and quit or run as a different user. When I select quit, it says Are you sure? [y/N] I don't see any files to be saved there. So where does it store firefox in RAM? I don't see any folder named firefox, chrome etc. I tried cd /usr/bin/firefox/ to view its source code, but cd /usr/bin/firefox/firefox-bin gives bash: cd: /usr/bin/firefox/firefox-bin: Not a directory Please someone clarify the situation? A: The default Ubuntu (and Debian) system-wide installation of Firefox will be in /usr/lib/firefox, whereas your own personal install will be at ~/.mozilla/firefox/blah.default. Most users keep there personal install of Firefox in their home directory, so that it would not impact other Firefox users (if there were Firefox users) on the system. Regarding your comment I think I have to kill firefox? (killall firefox) It depends, if the process is currently running it would need to be killed. You can kill any process by process name, or ID, by using pkill firefox pkill firefox killall firefox killall firefox The last example would only work if it was running on a shell that supports aliases. Example: If you have the firefox browser installed and running then ps -aux | grep -i firefox will show the PID of the firefox process. In the example above I used ps -aux for the first column and grep -i firefox for the second. Thus the last command would kill the firefox process based on its PID, if you were not sure which PID to kill, you could use ps -aux | grep -i firefox | awk '{print $2}' | head -n 1 or ps -aux | grep -i firefox | cut -d ' ' -f2 for example.
